Labyrinths
2020-12-19 #Python #C# #Unity

I had just come out of a course on algorithms with Valerie King, learning about algorithms like bfs, dfs, etc. when I wondered what a labyrinth made with each path finding algorithm would look like.

So I coded some of the algorithms in python, sprinkled in some randomness and printed them out.

DFS

Kruskals algorithm

Prims algorithm

As soon as I printed them out I realized that Kruskals and prims looked… similar, on reviewing my notes from class I realized that made sense, Kruskals made a minimum spanning forest and Prims a minimum spanning tree. Because a minimum spanning tree of a connected graph is just a minimum spanning tree, they were literally the same.

BFS algorithm

You may notice that this algorithm is not very good for making a maze, that is because it finds the shortest path between one point and every other point, so it creates easy mazes to traverse.

Game

After making these I wondered what it would be like to put this into a game, so I whipped up a demo where there are a series of blocks, each made with a randomly selected algorithm and some are just empty. I added lots of passageways between blocks so they could all connect and a few empty blocks for variety.

You can’t tell here but the maze procedurally generates rooms as you walk around, using random values seeded with block coordinates so that the world is consistent.
