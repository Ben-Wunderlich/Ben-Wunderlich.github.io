Pydoku
2020-11-14 #Python

My mom always has those giant sudoku books and goes through them like nothing. So as I was in the depth of the pandemic I decided to make a program to generate endless sudoku problems for her.

I did a bit of research and found an algorithm to generate them, but it looked non trivial to implement so I figured I would try on my own first. I picked the most basic possible algorithm: fill in random numbers and check if it is a valid sudoku board, if it is, remove numbers and print, if not? Just try again. This is not the most elegant algorithm, but I found it produced around 3 working boards per second, not a lot, but on the scale I was working with and the delay of sending a pdf to the printer it was just fine.

So I never bothered with the algorithm and worked with the PIL library to make a pdf template to put the numbers into, added answer pages, page numbers and boom, endless sudoku.
