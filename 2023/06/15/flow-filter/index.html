flow-filter
2023-06-15 #C++

I stumbled across a generative art tool, the first prompt I gave it was simply the word ‘sadness’, this is what I got.

I found this image to be wonderful and wanted to make a filter that could mimic this effect.

Polygon filter

Because of the cpu requirements for this kind of filter I knew I couldn’t use any of my existing frameworks in python or javascript. I knew C++ was fast enough and would have something I could use, so I looked around and found openFrameworks to be just what I was looking for. My C++ was a little rusty but the framework is easy to use and I was able to quickly implement my ideas.

My ideas

To match the original I wanted areas with less contrast to have less polygons, the way I did that was by starting with a single rectangle and using the following recursive method of splitting.

  1. check the contrast of pixels currently enclosed by the shape.
  2. if the contrast is too high, split into more shapes and start process again with new sub-shapes.

Results

I did a bit of parameter tuning and ended up with these results.

All my results were okay, but lacked the soul in the original so I moved on to try other methods.

Flow technique

I tried treating an image as the base of a flow system, using this method each pixel has a direction that is determined by the brightness of the pixel, then flows start periodically and trace their path following the angle at that pixel. You can learn about flow fields in more depth on my flow fields post.

Refining flow filtering

First I just had the angle be brightness value, which gave this.

As you can see, it looks like nothing. There is too much noise and there is no order to the pixel brightness.

Gradient ascent

To try and improve the results I changed it so instead of following flow field it would look at its neighbors and move towards the brightest spot. This technique is useful in many applications and is called gradient ascent.

Gradient descent

Adding depth

The last two attempts were neat, but lacked something because they were just two colors. I started making the color of the flow depend on the pixel that it ended on, first I had it in grayscale

Then in color.

Lastly I wanted more chaos so I added circular motion that dragged each flow loosely in a circle while they tried to find the light.

Because I like this method, here is it with a flower.

And for this one I added many more flows and removed the circular force. Using this many flows made the program take a while to run, but I think it was worth it.

Concentric circles

A year or so later I stumbled across this image on reddit and was reminded of my circular attempt. I decided to continue in a more radial direction inspired by that image, I wanted an image created by concentric circle segments.

Basic technique

For this technique I moved out in circles, each time checking the pixels that would be covered by a test arc and filling in an amount of that arc depending on the darkness of the test arc.

Adding complexity

The basic version was fine but not that interesting, so I tried adding colors from the image and adding more fine detail with smaller arcs and more concentric circles.

The concentric circle technique is fine but it could use some refining, if I ever pick it up again.
