neural-nets-start
2023-06-07 #Python #ML

At the moment, AI(also called Machine Learning or ML) is all the rage, and CS professors know this, so in many courses they insert AI even when it has nothing ot do with the course material. I didn’t really feel like I had a handle on it until I took a specific course on machine learning, it put each ML technique in context and provided the math of how they work.

graph search

Overall a great takeaway from the start of the course is what ML can be other than neural networks, namely graph searches. When you can represent a problem as a graph you can use the massive body of knowledge on how to deal with graphs to solve problems.

Many problems that may not seem like graphs can be modeled very well, for example tic tac toe can have each possible state of the game board as a node and two nodes are connected if one can transform into the other by a play move. Alone this takes up a lot of space, but you can make the graph as you go and only make the nodes you need to.

Logical agents

Another powerful ML technique is using the power of logic, literal logic, in its most pure form. Each program of this type is represented by clauses, for example you could have some clauses

  • if you hear the beast it is within 50m
  • if the creature is within 2m you lose
  • if you kill the beast you win
  • if there is nothing in a direction you can move there

Then you can use these clauses to prove that there is no solution or find solutions and prove that they are optimal.

Markov chain

Markov chains are another very powerful ML tool, they work by looking at a stream of information and trying to match that with what has happened in the sequence so far. For example a friend of mine used markov chains to extend songs. His program would start at the end of the song and try to see if the last 50 notes occurred somewhere else in the song, if they are, the note after those 50 is chosen to extend the song, if not it tries again with 49, and so on. You can also do this with a probabilistic approach, checking what the last few notes usually lead to and having those notes chosen with higher probability.

This approach can be adapted to many other problems and has many variations that can be fine tuned to each problem.

Neural networks

Neural nets are in premise very simple, there are a series of nodes that are connected, each passes on values to nodes it is connected to based on some values the node has stored and any values it was fed. At first these internal values are random, but by feeding in data in the first node(s) and changing the values so it is closer to the output you want, you can create shockingly useful programs.

Classification

One use of neural is feeding in an image and label that image in some way, this starts with one input node for each pixel, then more layers that each connect to the previous layer, eventually tapering down to output nodes that correspond to how many classes you want to separate the images into. This type of system can be harnessed in self driving cars, machine vision, cancer detection and so, so much more.

Classification is amazingly useful, and can also be used to generate something other than a label. This can be done by having an adversarial network, where a generator network creates the information, then classification is used to show how bad the result is, which then pushes the generator further in that direction. Under this technique the generator can get as good as the classifier can judge.
