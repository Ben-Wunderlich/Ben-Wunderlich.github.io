2017-04-27 #Unity #Javascript

The idea of game development was not new to me, I had dissected game files since I was a kid playing harry potter rpg games, but I had never given a thought to doing it myself.

I was watching twitch one day, looking for some new channels, when I stumbled into Markipliers stream. He mentioned that Unity was going just made free, and that you could use it to make games, and for whatever reason, I immediately knew I had to do it. I was still in chemistry at the time and had never written a line of code in my life, but I knew that I was going to make a game.

After installing, I messed around and created this using object creation, rotation, moving and scaling.

But this was not a game, and after looking through some forums I realized that to make a game I would need to use… code. I saw people were making code either in something called C# or something called Javascript, so naturally to decide which to use, I flipped a coin, which came up javascript.

Now I finally began my journey into programming, at first it was just lots and lots of looking through forum posts, copying lines and mashing them together without any knowledge on how any programming language worked, only what each line did, so I worked backwards from that to learn the language.

One funny thing I remember from this time was using integers to keep track of if a quest segment was active or not, having it be either 1 or 0. After doing most of the game like this I discovered this mystical object called a boolean, I just remember being so excited that there was something better than an integer for what I was doing.
