mspaint-program
2018-12-09 #Python #Art

I had just started learning how to code and wanted to do something with Python, so I looked through a list of Python libraries. What caught my eye was Pyautogui, a library for creating mouse and keyboard inputs from a script.

After messing around and making some programs to play video games, I thought it would be amusing to use microsoft paint with it. This became my first foray into generative art.

First I did basic geometric shapes, starting with a circle.

To my surprise it was not a circle, but some kind of spiral? This process of trying to make an image, failing but making something more interesting is a process I continue doing to this day.

These two are some of the basic geometry. As you can see the lines are not solid, this is because I was controlling the mouse with equations of the shape and just periodically marking the current value at a constant spacing.

An example of this is most clear in the circles, I am making some number of points for each circle, but as the circle gets bigger, the lines gets more uneven.

This could be solved with adjusting based on the radius, but it still falls apart in more complicated shapes where within one shape there is no single spacing amount that would always work. For now in the images below I dialed in the spacing manually.

some basic geometry

After the circles, triangles and squares I started looking for more interesting shapes, I found the description of some cool shapes on wikipedia like Maurer roses, Hypotrochoids and Lissajous curves. Others I just made up, often changing formulas from shapes I thought looked interesting, I called the below shapes after the first horlage because at first they looked like an hourglass.

Horlage

Roses

These were a lot of fun to make, but I wanted to make something more crazy. I had heard about fractals, mentioned as some very complicated shape, and thought I would try that. So I found some code from wikipedia on Julia sets and gave it a try.

It was so slow, I ended up only doing every tenth pixel and simplifying to black or white but even still the below image took around 30 minutes before I cut it off.

fractals

At the time I thought it was just because fractals were so complicated, but it was actually because I did not know how fractals worked, Knowing what I do now, I could have done the following two modifications to make it run in 2 seconds or less.

1. stop checking for interrupts

My programs took control of the mouse and keyboard and things often went very wrong, I broke my computer several times when the mouse clicked many things it should not have. To counteract this I added a hotkey that terminated the program, this was usually fine, but in the fractal code it was checking 1000 times per pixel which was just too much.

2. lower infinity

I will go into detail in later posts but instead of doing 1000 rounds of calculations per pixel I could have lowered it to about 30 and gotten the same result.

At the time, I thought it was slow just because I was using paint and thought it would be faster if I switched to making images directly with code instead of wasting time on mouse operations. Although this idea was incorrect, it pushed me to explore and discover the world of generative art.
