Internet-Protocols
2021-11-21 #Python

I was in a course on computer communication and we were learning all about the protocols that makes the internet possible. To make it a little different we were allowed to assume what responses would be when sending packets, this is less secure but can sharply reduce the number of packets sent. More marks would be awarded to the fastest protocols.

background stuff

First I’ll recap udp, tcp and HTTP, I am going to ignore everything below UDP(like IP).

what is UDP?

UDP simply takes some bytes of data and sends it over a network to an address.

and TCP?

TCP is similar to UDP except it can send a stream of bytes and also keeps track of whether messages are received successfully, resending them if it failed and keeping them in order.

HTTP on top

HTTP is built on top of UDP or TCP and deals with sending and receiving complete files along with response codes that can communicate anything that needs to be know by the other party.

what I did

making HTTP

First we had to recreate the HTTP protocol using tcp, this was a lot of parsing strings to check response codes, and managing multiple connections so everything doesn’t interfere with each other.

making TCP

Then we had to remake tcp using udp, this was where things got interesting, it was a lot more variable how you could approach the problem, and it needed a lot of code to check all the different ways messages could be received, lost, reordered at each part of the connection lifecycle.

The overall idea of my design was instead of sending back responses one at a time like many others did, I would have a buffer of messages to send, and as each packet is unpacked, add the responses and assume client responses, sending them all combined together once the packet is fully parsed. Because the task was to make as efficient as possible, Instead of going for simplicity I made lots of data structures to manage each stage of the connection and the data transmitted from and to the client. This decision made it more complicated but also made almost every operation O(1).

After everyone had submitted their assignments, they were tested and timed. My solution was the fastest so it came in first place!
