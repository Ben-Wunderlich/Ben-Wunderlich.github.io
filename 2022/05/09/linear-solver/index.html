linear-solver
2022-05-09 #Python

This is going to be a rough overview of an entire course on solving linear programs, taught by the legendary Bill Bird. The culmination of the course was creating our own program that can solve any linear system.

What is a linear program

A linear program consists of two parts,

Objective function.

an objective function is a formula each variable is fed into to produce an output value

Constraints.

a set of constraints that limit what values the variables can take

Here the objective function is 8x1+4x2+2x3 and each line below it constrains what value x1, x2 and x3 can take relative to each other.

how do you solve it

You can manipulate the above equation, solving for one constraint and substituting that everywhere else that variable appears. A wild fact that becomes very useful is that doing this algebraic manipulation is the same thing as traversing the edges of a multidimensional polygon with a number of dimensions equal to the number of variables. In this interpretation, each corner has coordinates that are values of each variable, and you can use that to find the current value gained by the objective function, and because they are linear programs, once you find a point you can keep going around edges until you find the point that has a maximum value.

things that can go wrong

This sounds great but there some problems.

how do you choose the next point?

When looking at the problem as finding a path along a shape, a huge question is which step to take when at a vertex. There is no single best technique, and different ones require more computation to make a better decision and some are better for different types of problems. Here I will use the algorithm “largest next increase” which calculates how much the objective value would be increased by each move and going the direction that increases it the most.

how do you find the first point?

Finding the next point is great, but first you need to find a single point to work with. If you set every value to 0 that will often find an initial(though bad) solution, but the problem comes when that point does not exist. This can be seen as the shape does not pass through the origin, to fix this you scale up the shape, relaxing constraints until it is valid at the origin, then once you can solve this you can scale it back down and use that as a point to start with.

what if there is no solution?

When finding the first point, if you scale the solution back down and that point is not valid, you know there must be no solution.

what about standing at the edge of infinity?

Imagine a single program with only the objective function x1 and constraint x1 0, you could just keep increasing x1 as large as you want. Here the shape is unbounded, so when moving around it you will eventually get to a place where the next point does not exist, this must be because the shape is unbounded.

What this means for the original problem is program dependant, it could mean the constraints were set up wrong, or there is a tax loophole that lets you create infinite money, or something completely different, it all depends on the problem the program was made to model.

what if it goes around in an infinite loop?

Because finding the largest next increase is only a heuristic, you could still find yourself going around in a circle. This can happen there is a flat face of the shape and the increase is 0 so all directions are the same. When this happens you can use the wonderfully named ‘bland’s rule‘ for finding the next point, always choosing the point with the lowest index, this will be slow but will eventually end, and once you are out of it you can continue the algorithm.

Fun fact about bland’s rule, it is both very bland, and named after its creator Robert G. Bland.

Duality

This is already so much material, but I simply must mention one of the most mind blowing things from the course, Duality. So far we have been working to try an maximize an objective value, but because math and the universe is insane, if you swap the columns and rows of the system of equations, you will create the dual of the problem which has a number of bizarre effects.

  1. instead of maximizing a value, following the algorithm outlined here will make the constraints less broken until it gets to the optimal value.
  2. the constraints of original and dual problems are both satisfied only at a single point, and at that point they are both optimal.

But wait there is more, there are many things other than linear programs where duals are a thing, some have known uses, but many remain useless, for now. These dual programs sometimes look completely different than the original, also which formulation of the problem is the dual and which is the original is simply a matter of perspective.

We can use duality to swap between the original and dual forms and use whichever is easier. An interesting thing about linear programs is you can represent the program really well with a matrix, then solve it by doing matrix operations, for example transposing it is the same as converting it to the dual problem.

The assignment

After learning all of this in great detail, and with stories of history and rigorous proofs of each premise above, we were tasked with using this knowledge to create a program that can solve any linear program. I was planning on going into details here about my implementation, but after a whole course of learning all this, the implementation was pretty easy because of how thorough my understanding of the topic was, and most of it was just done as described above.

In making the solver, one of the test cases we were given had over 200 variables and 200 constraints, it was quite a problem to run. Often in office hours people came asking about this case, one time someone came in say their program running for 4 seconds on it and asked if that was too long. Then the next person came in and asked if there was a time limit for testing, as their program had been running for over 3 hours on that test case.

This assignment and the course as a whole was so much fun and gave me some insight into how problems are converted into math and the techniques that can be created to solve them.
